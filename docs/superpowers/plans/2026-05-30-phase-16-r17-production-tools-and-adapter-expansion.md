## Goal

Ship the smallest production-grade local-daemon real-tool slice behind the existing `/tools/invocations` API: configured `fetch`, `web_search`, `github`, `repo`, and command-catalog `shell`, all deny-by-default, allowlist-gated, approval-gated by default, bounded, redacted, and fake/no-spend tested.

## Scope Challenge

1. Existing code already partially solves this through `packages/core/src/services/tool-router.ts`, `packages/core/src/services/local-policy-gate.ts`, `packages/core/src/services/approval-service.ts`, `packages/storage/src/sqlite/tool-invocation-store.ts`, `packages/protocol-rest/src/middleware-routes.ts`, and `apps/daemon/src/app.ts`. Extend those paths; do not create a second tool API or a runtime adapter path.
2. Minimum useful change: keep one route, keep `fake_echo`, replace hard-coded real-tool denial with an explicit real-tool policy, add adapters only for the five configured local tools, wire them in the daemon only, and document the boundary. `browser`, hosted tools, connected-node tools, generic process, PTY, dashboard, and TUI stay out.
3. Complexity smell: this is L and touches more than 8 files because contracts, policy, adapters, approval lifecycle, daemon wiring, REST tests, and docs all need to agree. The plan keeps task ownership disjoint and avoids new public surfaces. P16-T3 and P16-T5 were re-challenged for splitting; they remain cohesive package-level tasks because splitting either would create overlapping ownership of adapter exports/tests or the daemon/REST route contract, so the plan instead adds explicit constructor contracts and table-driven matrices to reduce implementation ambiguity.
4. Built-in check: use Node `URL`, `AbortController`, `fetch`, `dns/promises.lookup`, `net.isIP`, `path.resolve`, `path.relative`, `child_process.spawn`, `crypto.createHash`, existing Zod schemas, existing stores, existing EventBus, existing approval routes, and existing artifact content stores. Do not add URL parsers, HTTP libraries, process managers, browser automation libraries, PTY packages, or shell interpreters.
5. Distribution check: no new package, CLI binary, container, or hosted artifact ships. The existing packages build through `pnpm build`; OpenAPI JSON remains generated by `@switchyard/contracts`.

## Architecture

R17 keeps the public surface fixed. The route remains `POST /tools/invocations`; the body becomes strictly validated per tool type, but no `/exec`, `/shell`, `/process`, `/terminal`, `/pty`, `/sandbox`, `/browser`, or top-level/tool-search execution route is introduced. The existing middleware memory route `GET /memory/search` and operation id `searchMemory` remain valid and must be explicitly allowed by boundary tests.

```text
POST /tools/invocations
  -> REST envelope and type/input validation
  -> ToolRouter.invoke
  -> LocalPolicyGate.decideTool
       deny              -> persist denied invocation, emit bounded error
       approval_required -> persist queued invocation + pending approval
       allow             -> execute only if config explicitly allows no approval
  -> approval approve/reject/expire
  -> ToolRouter.resolveQueuedByApproval
  -> policy revalidation against stored immutable execution plan
  -> adapter.invoke(executionPlan)
  -> bounded artifacts + tool.result
```

Policy is deny-by-default at global, tool, and target levels. The default daemon config has `SWITCHYARD_REAL_TOOLS_ENABLED=0`, every per-tool enabled flag absent, an empty shell catalog, no GitHub token, and no search provider. When enabled, the only allowed placement is local. Hosted server, hosted worker, and connected node continue to deny or omit real-tool execution.

Required daemon environment names:

- Global: `SWITCHYARD_REAL_TOOLS_ENABLED`, `SWITCHYARD_REAL_TOOLS_ALLOWED_PLACEMENTS`, `SWITCHYARD_REAL_TOOLS_APPROVAL_DEFAULT`, `SWITCHYARD_REAL_TOOLS_APPROVAL_EXPIRES_MS`, `SWITCHYARD_REAL_TOOLS_MAX_CONCURRENT`, `SWITCHYARD_REAL_TOOLS_MAX_INPUT_BYTES`, `SWITCHYARD_REAL_TOOLS_MAX_INLINE_OUTPUT_BYTES`, `SWITCHYARD_REAL_TOOLS_MAX_ARTIFACT_BYTES`, `SWITCHYARD_REAL_TOOLS_DEFAULT_TIMEOUT_MS`
- Fetch: `SWITCHYARD_FETCH_TOOL_ENABLED`, `SWITCHYARD_FETCH_ALLOW_HOSTS`, `SWITCHYARD_FETCH_ALLOW_METHODS`, `SWITCHYARD_FETCH_ALLOW_CONTENT_TYPES`, `SWITCHYARD_FETCH_MAX_REDIRECTS`, `SWITCHYARD_FETCH_TIMEOUT_MS`, `SWITCHYARD_FETCH_MAX_RESPONSE_BYTES`, `SWITCHYARD_FETCH_ALLOW_HEADERS`
- Web search: `SWITCHYARD_WEB_SEARCH_TOOL_ENABLED`, `SWITCHYARD_WEB_SEARCH_PROVIDER`, `SWITCHYARD_WEB_SEARCH_BASE_URL`, `SWITCHYARD_WEB_SEARCH_API_KEY`, `SWITCHYARD_WEB_SEARCH_MAX_RESULTS`, `SWITCHYARD_WEB_SEARCH_TIMEOUT_MS`, `SWITCHYARD_WEB_SEARCH_MAX_RESPONSE_BYTES`
- GitHub: `SWITCHYARD_GITHUB_TOOL_ENABLED`, `SWITCHYARD_GITHUB_TOKEN`, `SWITCHYARD_GITHUB_ALLOW_REPOS`, `SWITCHYARD_GITHUB_TIMEOUT_MS`, `SWITCHYARD_GITHUB_MAX_RESPONSE_BYTES`
- Repo: `SWITCHYARD_REPO_TOOL_ENABLED`, `SWITCHYARD_REPO_GIT_BINARY`, `SWITCHYARD_REPO_ALLOW_CWD_PREFIXES`, `SWITCHYARD_REPO_MAX_PATHS`, `SWITCHYARD_REPO_TIMEOUT_MS`, `SWITCHYARD_REPO_MAX_OUTPUT_BYTES`
- Shell catalog: `SWITCHYARD_SHELL_TOOL_ENABLED`, `SWITCHYARD_SHELL_COMMAND_CATALOG_PATH`, `SWITCHYARD_SHELL_ALLOW_CWD_PREFIXES`, `SWITCHYARD_SHELL_TIMEOUT_MS`, `SWITCHYARD_SHELL_MAX_OUTPUT_BYTES`

Policy returns a redacted decision:

```ts
type ToolPolicyDecision =
  | { decision: "deny"; reasonCode: string; policyTrace: Record<string, unknown>[] }
  | { decision: "approval_required"; reasonCode: string; approvalType: ApprovalType; expiresAt: string; policyTrace: Record<string, unknown>[]; executionPlan: ToolExecutionPlan }
  | { decision: "allow"; reasonCode: string; policyTrace: Record<string, unknown>[]; executionPlan: ToolExecutionPlan };
```

Adapters receive only `executionPlan`, never the raw request body. Adapter outputs are summaries plus optional artifact candidates. ToolRouter owns persistence, status transitions, event emission, artifact writes, and one-shot approval resolution.
